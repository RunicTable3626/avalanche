# web/ — theavalanche.net (Hugo + Cloudflare)

The marketing/docs site. Hugo static site, deployed to Cloudflare via Wrangler.

## Build output is not tracked

`web/public/` is the Hugo build output and is **gitignored** (`web/.gitignore`).
Editing a source file under `web/content/` or `web/layouts/` does **not** update
`web/public/` — it must be regenerated by Hugo. There is nothing to commit for
`public/`; only the sources are tracked.

**Always rebuild Hugo before deploying**, or the live site ships stale content.

```bash
npm run dev      # hugo server -D — local preview with drafts
npm run build    # hugo --minify — regenerate web/public/
npm run deploy   # hugo --minify && wrangler deploy — rebuilds, then deploys
```

`npm run deploy` rebuilds for you. A bare `wrangler deploy` does **not** — it
would upload whatever stale `public/` happens to be on disk, so prefer the npm
script.

## Conventions

- Blog posts live in `web/content/blog/`; new posts start with `draft: true`
  (see `web/archetypes/`). Drafts are excluded from `build`/`deploy` and only
  appear under `hugo server -D`. Flip to `draft: false` to publish.
- The Android APK download link on `web/content/getting-started/sideload-android.md`
  points at a GitHub release asset and must be bumped to the new version each
  release (e.g. `releases/download/vX.Y.Z/app-release.apk`).
- No emoji in site content or docs — plain text.
